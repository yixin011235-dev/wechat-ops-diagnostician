# 公众号运营诊断（wechat-ops-diagnostician）

把一个号的后台导出 + 它的目标，变成一份方向级、目标相关的诊断 + 按杠杆排序的动作清单。**账号无关**：对哪个号跑就喂哪个号的数据和目标。

核心立场：这不是"数据分析"，是**归因 + 诊断引擎**。它的可信度建立在两件多数人跳过的事上——按来源归一化、以及区分信号与噪声、诚实声明盲区。判断逻辑全在 `references/diagnosis_framework.md`，每一步都对照它。

## 工作流

### 第 0 步 · 定位与目标（最重要，不能跳）
**先确认这个号属于哪类原型、背哪些目标。** 不确认就诊断，必然套错标尺（典型错误：给本地服务号套"全国触达"目标）。

读 `references/diagnosis_framework.md` 第 1–3 节，让运营方明确：
- 原型（五类之一：红圈大所号 / 垂类 IP 号 / 法律新媒体平台 / 区域中小所号 / 跨界新兴）
- 目标及权重（本地品牌 / 政企法顾背书 / 招聘 / C 端案源 / B 端法顾案源 / 思想领导）

信息不足就先问，别假设。这一步定下来，后面所有指标都对着它解读。

### 第 1 步 · 解析
跑 `scripts/parse_exports.py <导出文件或目录>`，把微信导出统一成 canonical 结构（契约见 `references/data_schema.md`）。脚本自动判别两种真实格式：内容/阅读趋势（OLE2 三块）和常读分布趋势（伪 HTML）。先看它打印的解析摘要，核对来源结构和体量是否合理。

```bash
python scripts/parse_exports.py /path/to/exports --out bundle.json
```

依赖：`pandas`、`xlrd`（读 legacy .xls）。缺则 `pip install xlrd --break-system-packages`。

### 第 2 步 · 归一化 + 自己的基线
绝对数字无意义。按来源、按内容类型归一化，跟这个号**自己**的滚动基线比；跨月看来源结构和受众结构的漂移。方法见 `diagnosis_framework.md` 第 7 节（信噪纪律）：小样本＝方向级结论、注意分母变化、近期加权。

### 第 3 步 · 横纵分层诊断
**横**：用第 0 步的原型，定位它的现实角色、能力上限、"好"的定义（第 2 节那张原型表）。**纵**：用数据看它自己走成了什么样。两者交汇，逐层体检：
- **触达**：靠哪台引擎（来源结构 → 引擎 → 杠杆，第 6 节映射）
- **受众**：谁在反复读（标注"是铁粉非全部受众"；体量、城市层级、年龄、性别）
- **内容组合**：什么读得多、什么有真实目标价值、长尾
- **转化**：私域出口在不在（第 5 节：2026 年公众号靠导流私域才有案源价值）

定位漏斗哪层在漏，归因到结构原因，别看到"阅读低"就笼统改标题。

### 第 4 步 · 对着目标评判
逐项判 working / 漏，**每项挂到它真正服务的目标**。按原型的现实角色判——对区域所号别拿天同当尺，低打开率、不爆款、规模小是赛道常态（第 4 节结构性现实），不是失败。

### 第 5 步 · 盲区声明
照 `diagnosis_framework.md` 第 9 节，明说这套数据答不了什么（无完读率/送达/转化、常读≠受众、城市只到层级、搜索读者地域未知…）。**盲区是一等输出**，是可信度的来源。

### 第 6 步 · 动作清单（按杠杆排序）
每条动作：修哪个漏、可回测。案源类目标几乎总指向同一杠杆——**建私域出口**（正文内企微来源码，见 `references/source_key_spec.md`）。涉及 CTA/引流/AI/数据采集的，附合规边界（第 8 节）。具体改法交棒：标题→wechat-title-expert、结构→wechat-writing-coach、涨粉/私域→personal-brand-system。

### 第 7 步 · （埋点上线后）转化归因
若有企微来源线索，按 `source_key_spec.md` 的来源键和内容数据 join，补上"哪条线/哪篇带来了咨询"。没有则照第 5 步标为最大盲区，并把"建埋点"列高优先级动作。

## 输出
严格按 `templates/report_template.md`：先划边界 → 一句话诊断 → 横纵分层体检 → 对目标评判 → 动作清单 → 重设成功标准。风格沿用一心惯例：判断句小标题、证据先行、不用"其实"和感叹号、克制排版。

## 边界与范围
- **它 owns**：测量、归一化/基线、归因、诊断、盲区声明、来源键 join。
- **它不碰**：单篇写作、标题优化、排版——交给手艺类 skill。
- **v1 范围**：单号、定期（如月度）跑、上述流水线。**先不做**：跨号横向对比、视频号/跨渠道归因、API 自动拉取、实时。先让它跑起来，再谈精密——别陷入"系统漂亮没人跑"。
